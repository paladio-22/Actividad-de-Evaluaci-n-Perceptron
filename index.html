Arquitectura del Perceptrón

Entradas
Salida
Pesos (w)

Función de Activación:

Función escalón unitario:

f(x) = 1 si x ≥ 0, 0 si x < 0

Algoritmo de Entrenamiento:

  1. Inicializar pesos y bias
  2. Para cada época:
    • Para cada muestra:
      • Calcular salida
      • Actualizar pesos si hay error
  3. Repetir hasta convergencia

Conclusiones y Aprendizajes

Hallazgos Clave
Recomendaciones
Limitaciones

El Perceptrón es efectivo para problemas linealmente separables, logrando precisiones superiores al 85%.

La tasa de aprendizaje afecta significativamente la velocidad de convergencia y el rendimiento final.

La selección de características es crucial; demasiadas pocas pueden reducir el rendimiento.

La implementación desde cero permite comprender los fundamentos del aprendizaje automático.

Para problemas simples: El Perceptrón básico es suficiente y eficiente.

Para convergencia rápida: Usar learning rate más alto con decaimiento.

Para mejor rendimiento: Realizar análisis de características antes de entrenar.

Para problemas complejos: Considerar Perceptrón Multicapa o SVM con kernel.

Linealidad: Solo puede resolver problemas linealmente separables.

Sensibilidad a características: El rendimiento depende en gran medida de la selección de features.

Convergencia: Puede no converger si los datos no son linealmente separables.

Capacidad limitada: Modelos más complejos pueden ofrecer mejor rendimiento en problemas no lineales.
